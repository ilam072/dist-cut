# Инструкция по запуску `dcut`

### 1. Подготовка

1. Собираем бинарник (в корне проекта, где лежит `cmd/main.go`):

```bash
make build
```

2. Создаём тестовый файл `test.csv` (пример):

```text
1,2,3
2,4,6
3,6,9
4,8,12
5,10,15
6,12,18
7,14,21
8,16,24
9,18,27
10,20,30
```
---

### 2. Запуск воркеров

Открываем два отдельных терминала (или больше, если есть ноды) и запускаем воркеры:

**Терминал 1:**

```bash
./dcut -port=8002
```

**Терминал 2:**

```bash
./dcut -port=8003
```

Каждый воркер должен вывести:

```
worker listening on 127.0.0.1:800X
```

Воркеры остаются живыми и слушают HTTP-запросы на `/process`.

---

### 3. Запуск координатора

В отдельном терминале запускаем координатор и передаём данные через stdin:

```bash
./dcut -coord -port=8001 -peers=127.0.0.1:8002,127.0.0.1:8003 -f=1,3 -d=, < test.csv
```

* `-coord` — запуск координатора
* `-port=8001` — порт координатора
* `-peers=127.0.0.1:8002,127.0.0.1:8003` — список воркеров
* `-f=1,3` — выбираем поля 1 и 3
* `-d=,` — разделитель

Координатор подключается к воркерам, распределяет шардированные строки, ждёт quorum (по умолчанию репликация 2, quorum 2) и выводит результат в stdout.

---

### 4. Пример вывода

Для приведённого `test.csv` вывод будет примерно таким:

```
1,3
4,12
7,21
10,30
2,6
5,15
8,24
3,9
6,18
9,27
```

> Обратите внимание: порядок может отличаться от оригинального `cut` из-за шардирования (каждый шард собирается отдельно).

---

## Сравнительный тест с оригинальным `cut`

Создаём эталонный файл через системный `cut`:

```bash
cut -d',' -f1,3 test.csv > expected.out
```

Запускаем `dcut` и сохраняем результат:

```bash
cat test.csv | ./dcut -coord -port=8001 -peers=127.0.0.1:8002,127.0.0.1:8003 -f=1,3 -d=, > actual.out
```

Сравниваем:

```bash
diff -u expected.out actual.out
```

* В `diff` строки могут быть переставлены из-за шардирования, но **содержимое полей совпадает**
* Фактически, функциональность `cut` сохранена, распределённая версия лишь не гарантирует исходный порядок строк.

---

## Флаги
```
-peers: список адресов peer в формате host:port через запятую (другие ноды)
-port: локальный порт для прослушивания
-coord: запуск координатора (читает stdin и распределяет задачи)
-rep: фактор репликации (сколько нод получает каждый шард)
-quorum: количество успешных ответов, необходимых для каждого шарда
-f: список полей (как в cut -f)
-d: разделитель (по умолчанию табуляция)
```
---

## Makefile
### Команды
* `make build` — собирает исполняемый файл `dcut` из `cmd/main.go`.
* `make clean` — удаляет собранный бинарник и файлы результатов тестов (`expected.out`, `actual.out`).
